You type `claude` and hit enter. What happens next?

You're sitting in your terminal, inside your project directory. No browser tab, no IDE plugin, no setup wizard. Just a prompt. You type: "The user signup test is failing. Fix it." And then you watch.

### The Agentic Loop

Claude reads the test file. Finds the related handler. Traces the validation logic. Edits the source. Runs the test suite. Sees a new failure. Edits again. Runs again. Green.

That's the loop: read, think, act, verify, repeat. You didn't point it at any files. You didn't approve each step. You described the goal and it worked backward from there.

Here's what makes this powerful -- Claude verifies its own work. Give it a test suite, a linter, a type checker, and the quality of output jumps dramatically. Without verification, it's guessing. With it, it's iterating.

### Permission Modes as Trust Levels

You wouldn't hand a new contractor the production keys on day one. Permission modes work the same way.

Cycle through them with **Shift+Tab**. **Default** asks before every edit and command. **Accept Edits** auto-approves file changes but still gates shell commands. **Plan** is read-only -- perfect for exploration. **Auto** lets a background classifier handle approvals for long-running tasks.

Which mode do you start a new task in? Default. Once you trust the direction, escalate to Accept Edits. Use Plan when you're still figuring out what to build.

### CLAUDE.md as Project Memory

Without a `CLAUDE.md`, Claude re-learns your preferences every single session. Your package manager, your test command, your file conventions -- all rediscovered from scratch.

Run `/init` and Claude analyzes your project, drafting a starting point. Then add what it can't discover on its own: why you chose that ORM, which directories are off-limits, what your team expects in a PR. That human context is what makes the memory file valuable.

### Subagents as Delegation

You need to understand how the billing module works before refactoring it. If you ask Claude directly, fifty files of research land in your conversation. Now your context is polluted.

Instead, Claude spins up an **Explore** subagent -- read-only, fast, isolated. It searches freely in its own context and reports back a summary. Your main conversation stays clean. You can also define custom subagents in `.claude/agents/` for specialized roles: a security reviewer, a docs writer, whatever your workflow demands.

### Git Integration

Ask Claude to commit your changes and it doesn't just run `git commit`. It reads the diff, understands what changed semantically, and writes a meaningful message. Branches, PRs, the whole flow -- native. For batch work, `/batch` splits a large change across parallel worktrees, one agent per unit, each opening its own PR.

### Notes

- The agentic loop is read-think-act-verify -- give it a way to check its work and quality jumps
- Permission modes are trust levels: start restrictive, escalate as confidence grows
- `CLAUDE.md` is persistent memory -- `/init` generates it, you refine it with human context
- Subagents keep research isolated from your main conversation
- Git integration is semantic, not a shell wrapper
- Headless mode (`claude -p`) unlocks CI pipelines and scripted automation
