You're typing a function and the next three lines appear in gray. You press Tab. The code lands, it's correct, and you keep moving.

That moment -- the seamless, zero-friction suggestion -- is the Copilot experience. But it's also just the beginning. Copilot spans from single-line completions all the way to autonomous agents that open PRs while you're at lunch.

### Inline Completions

This is where most people start, and where Copilot feels most invisible. You type a few characters, it suggests the rest. Accept with `Tab`, reject by keeping typing.

But it goes beyond autocomplete. **Next Edit Suggestions** predict where your next change will be -- not just at the cursor, but elsewhere in the file. You're renaming a variable, and Copilot offers to update the other references before you get there. Think of it as a pair programmer who reads ahead.

### Chat

When you need more than a suggestion, open Copilot Chat. Ask questions about your code, get explanations, brainstorm approaches. It works in the IDE, on GitHub.com, on mobile, and in the CLI.

Scope the conversation with participants like `@workspace` (VS Code) or `@project` (JetBrains) to give Copilot your full project as context. The difference between a vague answer and a precise one is usually context. Are you using these scope participants, or just chatting without them?

### Agent Mode

Here's where Copilot enters the same territory as Claude Code and OpenCode. Open Copilot Edits (`Cmd+Shift+I`), switch to Agent mode, and describe what you want.

Copilot figures out which files to change, makes the edits, runs terminal commands, checks the results, and iterates. The key difference from Edit mode: you don't pick the files. The agent does. You describe the goal and it works out the plan.

### The Coding Agent

What if the agent didn't need your editor at all? The **coding agent** works asynchronously on GitHub. Assign an issue to `@copilot`, and it spins up its own environment via GitHub Actions, implements the change, and opens a draft PR.

You review the PR like any other. Leave comments and the agent iterates. It runs CodeQL for security, scans for secrets, checks dependencies -- all built in. This is the "fire and forget" workflow.

### Spaces and Code Review

**Spaces** let you curate context -- bundle code, docs, specs, and design notes into a named collection. When you chat with Copilot in a Space, answers are grounded in exactly the material that matters. Think of Spaces as a reading list for your AI.

Copilot also **reviews pull requests**. Request a review from Copilot the same way you would from a teammate. It leaves suggestions, flags issues, and catches things humans miss on a Friday afternoon.

### Notes

- Inline completions with Next Edit Suggestions predict changes across the file, not just at the cursor
- Chat supports scoped context via `@workspace` and `@project` participants
- Agent mode plans and executes multi-file changes autonomously inside the editor
- The coding agent works asynchronously on GitHub -- assign an issue, review the PR later
- Spaces curate context for more grounded answers
- Built-in code review catches what tired eyes miss
- Choose Copilot when your workflow already lives on GitHub
