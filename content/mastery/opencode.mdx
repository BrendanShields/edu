What if you could swap the brain without changing the body?

One line in your config changes the model from GPT-4 to Claude to a local model running on your machine. Same interface, same workflow, same muscle memory. The tool stays constant. The intelligence behind it becomes your choice.

### The TUI

OpenCode's terminal interface isn't just a prompt in a box -- it's a full application. Fuzzy file search with `@`, inline shell commands with `!`, slash commands, agent switching with `Tab`.

Reference a file with `@` and OpenCode pulls it into context. Prefix a command with `!` to run it without leaving the conversation. Type `/undo` and it reverts the code change *and* removes the message from history. Why a TUI instead of an IDE extension? Portability. It works the same on any machine, any OS, any terminal.

### Provider Flexibility

This is where OpenCode separates itself. Anthropic, OpenAI, Google, local models through Ollama -- bring your API key and go. No vendor lock-in, no single point of failure.

New to LLM providers? **Zen** is OpenCode's curated gateway. Run `/connect`, select "opencode", and sign in. Zen offers a benchmarked list of models tested specifically for coding agent quality. Good defaults without the research. Or bring your own key with `opencode auth login` and choose exactly what you want.

### The Agent System

OpenCode ships with two primary agents and two subagents. Primary agents are your working modes. Subagents are specialists you call in.

**Build** is the default -- full tool access for reading, writing, and running commands. **Plan** is read-only, analyzing and suggesting without touching anything. Switch between them with `Tab`.

For delegation, invoke subagents with `@`. **General** handles multi-step tasks with full tool access. **Explore** is fast, read-only codebase search. Ask `@explore How does the payment module work?` and it digs through your code without modifying a thing. Have you tried delegating research to Explore while you keep working in Build?

### Zen Mode

When you don't want to think about providers or configuration, Zen mode strips everything down. Curated models, simplified setup, no API key juggling. It's the on-ramp for teams that want to start fast and optimize later.

### When to Choose OpenCode

Choose OpenCode when control is the priority. Open source means you can read the code, fork it, contribute. Provider flexibility means no single vendor owns your workflow. The TUI means it works everywhere.

If you need tight GitHub integration, look at Copilot. If you want the deepest Anthropic integration, look at Claude Code. But if you want to own every layer of your toolchain, OpenCode is the answer.
