Every time the agent edits a file, you manually run the formatter. Every time. You watch Claude write perfectly functional code with inconsistent indentation, and you open a terminal, run Prettier, and go back. What if it happened automatically?

Hooks make it automatic. And unlike instructions in `CLAUDE.md` -- which the agent may or may not follow -- hooks are deterministic. They always fire when triggered.

### Hook Events

A hook is a shell command tied to an event in Claude Code's lifecycle. Four events cover most use cases:

**PreToolUse** fires before a tool call. **PostToolUse** fires after. **Stop** fires when Claude finishes responding. **Notification** fires when it needs your attention.

Each hook has a `matcher` -- a regex that determines which tool triggers it. A `PostToolUse` hook with matcher `Edit|Write` fires only when Claude edits or writes a file, not when it reads or runs a command.

### Auto-Format: The Most Common Hook

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "jq -r '.tool_input.file_path' | xargs npx prettier --write"
          }
        ]
      }
    ]
  }
}
```

Add this to `.claude/settings.json`. Every file Claude edits gets formatted immediately, without Claude spending tokens on formatting decisions. How much time do you spend manually reformatting agent output right now?

### Blocking Dangerous Actions

`PreToolUse` hooks can block actions entirely. Exit code `2` means "stop -- don't do this." The agent receives your error message as feedback and adjusts its approach.

Want to prevent Claude from touching `.env`, `package-lock.json`, or anything in `.git/`? Write a bash script that checks the file path and exits with `2` if it matches a protected pattern. Exit code `0` means proceed. Simple, reliable, deterministic.

### Where Hooks Live

Three locations, from broadest to narrowest scope:

- `~/.claude/settings.json` -- all your projects
- `.claude/settings.json` -- single project, committable
- `.claude/settings.local.json` -- single project, gitignored

Use `/hooks` in Claude Code to browse what's configured.

### Headless Mode

Everything above works interactively. But Claude Code also runs without a human:

```bash
claude -p "Run tests and fix any failures" \
  --allowedTools "Bash,Read,Edit"
```

This unlocks CI pipelines, automated migrations, and scheduled maintenance. Hooks still fire in headless mode -- the agent writes code, hooks enforce formatting and safety, and CI validates the result. A fully automated coding pipeline with guardrails baked in.

### Notes

- Hooks are deterministic shell commands tied to agent lifecycle events -- they always fire
- Four events: PreToolUse, PostToolUse, Stop, Notification
- Auto-format with a PostToolUse hook on `Edit|Write` -- the most common use case
- Block dangerous actions with PreToolUse hooks that exit with code `2`
- Three config locations: global, project (committable), project-local (gitignored)
- Headless mode (`claude -p`) enables CI pipelines with hooks still enforcing guardrails
