## Inline Suggestion Shortcuts (VS Code)

These shortcuts control the autocomplete-style code suggestions that appear as you type.

### macOS

| Action | Shortcut | Command Name |
|--------|----------|--------------|
| Accept suggestion | `Tab` | `editor.action.inlineSuggest.commit` |
| Dismiss suggestion | `Esc` | `editor.action.inlineSuggest.hide` |
| Show next suggestion | `Option+]` | `editor.action.inlineSuggest.showNext` |
| Show previous suggestion | `Option+[` | `editor.action.inlineSuggest.showPrevious` |
| Trigger suggestion | `Option+\` | `editor.action.inlineSuggest.trigger` |
| Open suggestions pane | `Ctrl+Return` | `github.copilot.generate` |
| Toggle Copilot on/off | No default | `github.copilot.toggleCopilot` |

### Windows / Linux

| Action | Shortcut | Command Name |
|--------|----------|--------------|
| Accept suggestion | `Tab` | `editor.action.inlineSuggest.commit` |
| Dismiss suggestion | `Esc` | `editor.action.inlineSuggest.hide` |
| Show next suggestion | `Alt+]` | `editor.action.inlineSuggest.showNext` |
| Show previous suggestion | `Alt+[` | `editor.action.inlineSuggest.showPrevious` |
| Trigger suggestion | `Alt+\` | `editor.action.inlineSuggest.trigger` |
| Open suggestions pane | `Ctrl+Enter` | `github.copilot.generate` |
| Toggle Copilot on/off | No default | `github.copilot.toggleCopilot` |

## Chat Variables (VS Code)

Type `#` in the chat prompt to include specific context.

| Variable | What It Includes |
|----------|-----------------|
| `#block` | Current block of code |
| `#class` | Current class |
| `#comment` | Current comment |
| `#file` | Current file's content |
| `#function` | Current function or method |
| `#line` | Current line of code |
| `#path` | File path |
| `#project` | Project context |
| `#selection` | Currently selected text |
| `#sym` | Current symbol |

## Chat Participants (VS Code)

Type `@` to invoke a domain expert. Each participant has specialized knowledge.

| Participant | Description |
|-------------|-------------|
| `@workspace` | Has context about your project structure, code interactions, and design patterns |
| `@terminal` | Has context about the VS Code terminal and its contents |
| `@vscode` | Has context about VS Code commands and features |
| `@github` | Uses GitHub-specific Copilot skills (search, PRs, issues) |
| `@azure` | Context about Azure services (public preview) |

### References in Visual Studio

Use `#` followed by file names or keywords:

| Example | Description |
|---------|-------------|
| `#MyFile.cs` | Reference a specific file |
| `#MyFile.cs #Other.cs` | Reference multiple files |
| `#MyFile.cs: 66-72` | Reference specific lines |
| `#solution` | Reference the current solution |

## Copilot Edits Modes

### Edit Mode (VS Code, JetBrains)

You manually select which files Copilot can modify. Copilot proposes changes, and you accept or reject each edit. Best for quick, targeted updates where you want full control.

### Agent Mode (VS Code, Visual Studio, JetBrains)

Copilot autonomously determines which files to change. It proposes code changes and terminal commands, then iterates to fix issues. Best for complex, multi-step tasks.

**Agent mode capabilities:**
- Determines relevant files automatically
- Proposes file edits as diffs
- Suggests and runs terminal commands (with your approval)
- Iterates to fix errors and complete the task
- Supports MCP server integration for external tools

## Copilot Coding Agent

An autonomous agent that works on GitHub issues and creates PRs. Available in Copilot Pro+, Business, and Enterprise.

**How it works:**
1. Assign a GitHub issue to Copilot (or ask in Copilot Chat)
2. The agent works independently on a branch
3. It creates a pull request for your review
4. You review and merge like any other PR

## Custom Instructions

### Repository-Wide Instructions

Create `.github/copilot-instructions.md` in your repository:

```markdown
# Copilot Instructions

- Use TypeScript for all new files
- Follow the existing patterns in src/utils/
- Write tests using Vitest, not Jest
- Use functional components with hooks for React
```

### Path-Specific Instructions

Create files in `.github/instructions/` with the `.instructions.md` extension:

```markdown
<!-- .github/instructions/api.instructions.md -->
---
applyTo: "src/api/**"
---
- All API handlers must validate input with Zod
- Return consistent error response format
- Add OpenAPI documentation comments
```

### Organization Instructions

Organization owners can set instructions that apply across all repositories in the organization. Configured through GitHub organization settings.

### Personal Instructions

Individual users can set personal instructions that apply to all their Copilot interactions. Configured through GitHub personal settings.

### Agent Instructions

The Copilot coding agent also reads `AGENTS.md`, `CLAUDE.md`, or `GEMINI.md` files in the repository root for additional context.

### Custom Instructions Support Matrix

| Context | Chat | Coding Agent | Code Review |
|---------|------|-------------|-------------|
| Personal instructions | Yes | No | No |
| Repository-wide (`.github/copilot-instructions.md`) | Yes | Yes | Yes |
| Path-specific (`.github/instructions/`) | No | Yes | Yes |
| Organization instructions | Yes | Yes | Yes |
| Agent files (AGENTS.md, CLAUDE.md) | No | Yes | No |
