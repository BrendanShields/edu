# Managing Context

You know context is exhausted when the agent starts forgetting things you told it ten minutes ago.

It re-reads a file it already read. It tries an approach you already rejected. It stops following a convention you established at the start of the session. Responses get vaguer, more generic, less useful.

This isn't a bug. It's a full desk. And the fix is knowing when and how to clean it up.

### Clear between tasks

Old context from Task A actively hurts Task B. The agent is still holding files, outputs, and decisions from the previous task, leaving less room for the current one.

| Tool | Command | What it does |
|------|---------|-------------|
| **Claude Code** | `/clear` | Resets context entirely |
| **Claude Code** | `/compact` | Summarizes conversation, preserves key decisions |
| **OpenCode** | `/new` or `/clear` | Starts a fresh session |
| **OpenCode** | `/compact` | Compacts the current session |
| **GitHub Copilot** | New thread | Each thread is separate context |

When you switch topics, start fresh. It takes five seconds and saves twenty minutes of confused output.

### Offload research to subagents

Subagents are the best tool for keeping context clean. They run in their own context window, do their research, and return just a summary.

```text
# Claude Code
Use subagents to investigate how our auth system handles
token refresh, and whether we have existing OAuth utilities.
```

```text
# OpenCode
@explore How does the payment module handle refunds?
```

The subagent reads dozens of files. Your main context gets a paragraph-long summary. The research doesn't bloat your session.

### Store conventions, not conversation

Don't repeat project rules every session. Write them down once.

**Claude Code** reads `CLAUDE.md`. **OpenCode** reads `AGENTS.md` (and `CLAUDE.md` for compatibility). **Copilot** reads `.github/copilot-instructions.md`.

These files load at the start of every session and survive compaction. Keep them lean -- under 200 lines. If removing a line doesn't cause mistakes, delete it.

Run `/init` in Claude Code or OpenCode to auto-generate a starter file.

### When things go wrong

After two failed corrections on the same issue, stop correcting. Run `/clear` and write a better prompt from scratch. Arguing with an agent that has lost the thread wastes context making things worse.

If auto-compaction triggers, follow up with `/compact focus on [what matters]` to guide what gets preserved. The default compaction is smart but not perfect.

And if you find yourself in a session that's been going for an hour with dozens of exchanges? Start a new one. Carry forward the key decisions in your instruction file. A clean desk beats a cluttered one.
