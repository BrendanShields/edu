# Understanding Context

If you learn one concept from this workshop, make it this one.

Every AI coding tool has a hard limit on how much it can hold in its head at once. This limit is called the context window, and it determines everything about how well the agent performs.

### The desk metaphor

Think of the context window as a desk. Everything the agent is working with has to fit on this desk: your conversation, every file it's read, every command output, its own instructions, even the definitions of its tools.

A clean desk with the right documents lets you work brilliantly. A desk buried under every piece of paper from the entire project makes it hard to find anything.

What happens when the desk is full and you need to add something new? Something old has to go. The agent starts forgetting things -- instructions you gave earlier, files it read at the start, decisions you made together.

That's context exhaustion. And it's the #1 reason AI coding sessions go sideways.

### How context fills up

Not all actions cost the same.

Reading a small file? Cheap. A short prompt and response? Cheap. A grep that returns a few matches? Cheap.

Reading a 2,000-line file? Expensive. Running a test suite and dumping the full output? Very expensive. Having twenty back-and-forth exchanges? Expensive. Connecting multiple MCP servers that each add their tool definitions to every request? Surprisingly expensive.

Here's a rough sense of scale. Claude Code gives you 200K tokens by default, up to 1M with extended context. OpenCode depends on your model -- GPT-4o gets ~128K, Claude Sonnet ~200K, Gemini over 1M. Copilot varies by model and uses workspace indexing to stay efficient.

A token is roughly three-quarters of a word. So 200K tokens sounds like a lot -- until a single large file read eats 5,000 of them and a verbose test output eats 10,000 more.

### What happens when context runs out

The agent doesn't crash. It degrades.

First, it starts forgetting instructions you gave at the beginning of the conversation. It might stop following a convention you mentioned in message two.

Then it starts repeating work. It reads a file it already read. It tries an approach it already tried.

Eventually, responses get generic. The agent loses the thread of your specific problem and falls back to general patterns. It makes mistakes it wouldn't have made twenty minutes ago.

This is the worst kind of failure because it's subtle. The agent doesn't tell you it's running out of room. It just gets quietly worse.

### How each tool manages context

**Claude Code** auto-compacts when context hits ~95% capacity. It clears older tool outputs and summarizes the conversation. You can also run `/compact` manually, with an optional focus: `/compact focus on the API changes`. Run `/context` to see a visual map of what's using space.

Critically, Claude Code uses subagents that get their own fresh context window. Work they do doesn't bloat your main session -- they return just a summary.

**OpenCode** auto-compacts by default too. Configure it in `opencode.json` with the `compaction` settings. Press `Ctrl+X` then `C` to compact manually. The Explore and General subagents (`@explore`, `@general`) run in isolated context, just like Claude Code's subagents.

**GitHub Copilot** takes a different approach. It indexes your entire repository in the background, creating embeddings that let it find relevant code without loading everything. Use `@workspace` to search this index. Use `#file` or `#selection` to precisely control what enters context. Start new threads with `/clear` or `/new`.

### How to manage context well

You've seen how the tools handle mechanics. Your job is strategy.

Start focused. Point the agent at the specific files relevant to your task. Let it discover more as needed. "Look at `src/auth/session.ts` and fix the token refresh bug" is much better than "read everything in `src/` and tell me about it."

Break large tasks into sessions. Investigation in session one. Implementation in session two. Testing in session three. Each session starts with a clean desk.

Use persistent instructions instead of repeating yourself. Put conventions in CLAUDE.md, AGENTS.md, or `.github/copilot-instructions.md`. These files load at the start of every session and survive compaction. Keep them lean -- under 200 lines. Every line takes context away from your actual work.

And when the agent starts forgetting things? Don't fight it. Compact, or start fresh. A clean desk beats a cluttered one every time.
