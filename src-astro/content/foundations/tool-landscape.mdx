# The Tool Landscape

Every tool in this workshop follows the same agentic loop. Read, think, act, verify. But the philosophy behind each one is different, and that philosophy shapes everything about how you work with it.

Understanding the philosophy matters more than memorizing feature lists. Features change every week. Philosophy doesn't.

### Claude Code: the terminal believer

Claude Code comes from Anthropic's belief that the terminal is where real work happens.

You type `claude` in a project directory and start talking. The agent reads your files, edits your code, runs your tests, and manages git -- all without leaving the terminal. It has full access to your filesystem and shell, which means it can do anything you can do from the command line.

The tradeoff is intentional: Claude Code only works with Anthropic's models. No GPT, no Gemini, no local models. Anthropic built the tool specifically around Claude's reasoning capabilities, including extended thinking and a deep understanding of large codebases.

Think of it as a vertically integrated experience. One model, one tool, optimized end to end. If you're comfortable in the terminal and want the deepest agentic capabilities, this is the tool that goes furthest.

It also works inside VS Code, JetBrains, a desktop app, and on the web at claude.ai/code. But the terminal is home.

### OpenCode: the open door

OpenCode comes from a different belief: you should be able to use any model from any provider.

It connects to over 75 providers via the AI SDK. Anthropic, OpenAI, Google, local models running on your machine -- whatever you prefer. The OpenCode Zen program even curates a tested list of recommended models.

The interface is a beautiful terminal UI with panels, syntax highlighting, and session management. It feels polished in a way that surprises people for an open-source project.

What makes OpenCode philosophically distinct is its extensibility. You can create custom agents via JSON config or markdown files. You can define custom commands with template variables. You can share sessions via opncd.ai links. The project is fully open source, so you can inspect how it works, modify it, or contribute.

If you want freedom to choose your model, prefer open-source tools, or need to run local models behind a firewall, OpenCode is built for you.

### GitHub Copilot: the embedded assistant

Copilot comes from GitHub's belief that AI should be woven into the place you already work.

It lives inside your IDE. VS Code, JetBrains, Xcode, Visual Studio -- wherever you write code. And it starts with something the other two tools don't offer at all: inline code completions as you type. No prompt needed. Just keep typing and Copilot suggests the next line, the next function, the next block.

That inline experience is the fastest feedback loop in AI coding. But Copilot also has Chat mode for conversations and Agent mode for autonomous multi-step tasks. Its coding agent can even work on GitHub issues asynchronously, creating branches and pull requests while you sleep.

The tradeoff? You're inside the GitHub ecosystem. That's powerful if you're already there -- issues, PRs, code review, Actions all integrate seamlessly. It's limiting if you prefer terminal-first workflows or want deep agentic autonomy.

### Where they overlap

All three tools can read files, edit code, run shell commands, search codebases, integrate with MCP servers, and follow custom instructions. They all manage context windows and support session persistence.

The core agentic capabilities are converging. What differs is the surface.

### Where they diverge

The real differences come down to four things.

**Autonomy.** Claude Code offers the most granular control with six permission modes. OpenCode provides flexible per-tool allow/ask/deny rules. Copilot relies on IDE-integrated review and inline acceptance.

**Models.** Claude Code is locked to Anthropic. OpenCode works with anything. Copilot offers a curated selection including GPT-4o, Claude, and Gemini.

**Interaction.** Copilot is the only tool with inline completions -- the fastest, lowest-friction AI interaction. Claude Code and OpenCode are conversation-first, which is better for complex multi-step tasks.

**Ecosystem.** Copilot is embedded in GitHub. Claude Code integrates with Anthropic's broader platform. OpenCode is standalone and portable.

### Choosing -- or combining

There is no single best tool. But there is a best tool for a given moment.

If you live in the terminal and want deep agentic capabilities: Claude Code. If you want model flexibility or prefer open source: OpenCode. If you live in your IDE and love inline completions: Copilot.

Many developers use two tools together. Copilot for inline completions while typing, and a terminal agent for bigger tasks. The skills you learn in this workshop transfer across all three -- that's the whole point.
