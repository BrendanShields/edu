MCP is like USB for AI -- a universal way to connect tools to capabilities. Instead of every AI tool building its own Sentry integration, its own database connector, its own GitHub client, MCP defines one protocol that works everywhere.

One server, any client. That's the idea.

### What becomes possible

With MCP servers connected, your AI tool can reach beyond your codebase. Query a PostgreSQL database for user analytics. Create issues on GitHub. Investigate errors in Sentry. Pull up designs from Figma. Search documentation with Context7.

The agent already knows how to read, think, and act. MCP just gives it more to work with.

### Adding a server to Claude Code

Claude Code manages MCP servers through the CLI. A remote HTTP server is the simplest to set up:

```bash
claude mcp add --transport http notion https://mcp.notion.com/mcp
```

For local servers that need environment variables:

```bash
claude mcp add --transport stdio --env SENTRY_TOKEN=xxx sentry \
  -- npx -y @sentry/mcp-server
```

Manage your servers with `claude mcp list` and `claude mcp remove`.

### Adding a server to OpenCode

OpenCode configures MCP in `opencode.json`:

```json
{
  "mcp": {
    "context7": {
      "type": "remote",
      "url": "https://mcp.context7.com/mcp"
    }
  }
}
```

Remote servers with OAuth, local stdio servers -- both work. The format is different from Claude Code, but the concept is identical.

### Adding a server to Copilot

Copilot supports MCP in VS Code and JetBrains. Configure servers in your editor settings. The GitHub MCP Server is a first-party integration that works remotely -- no local setup required.

### Sharing servers with your team

Here's where MCP gets really useful. In Claude Code, create a `.mcp.json` file at the project root and commit it:

```json
{
  "mcpServers": {
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": { "DATABASE_URL": "${DATABASE_URL}" }
    }
  }
}
```

Environment variables expand with `${VAR}` syntax. Every teammate who clones the repo gets the same MCP tools automatically. No setup instructions. No "did you configure the database server?"

### Scoping and security

You might wonder: should I give the AI access to my production database? Probably not.

Claude Code supports three scopes: **local** (personal, not shared), **project** (committed to the repo), and **user** (global across all projects). Put sensitive servers in local scope. Put team-shared servers in project scope.

### Try it

Add the Context7 MCP server to your tool of choice. Then ask: "Look up the docs for [a library you use] using context7 and summarize the getting-started guide." Watch the agent reach outside your codebase for the first time.
