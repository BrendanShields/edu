Skills let you package domain knowledge so the agent doesn't start from scratch. Instead of explaining your deployment process every time, you write it once and invoke it with `/deploy`.

Think of skills as recipes. The agent has the cooking ability. Skills give it the recipe for your specific dish.

### How skills differ from memory files

Memory files (`CLAUDE.md`, `AGENTS.md`) load every session. Skills load on demand -- when you invoke them, or when the agent decides they're relevant. This keeps your context clean. A deployment skill doesn't need to sit in memory while you're debugging a CSS issue.

### Creating a skill

In Claude Code, skills live in `.claude/skills/`. Each skill is a directory with a `SKILL.md` file:

```yaml
---
name: fix-issue
description: Fix a GitHub issue by number
disable-model-invocation: true
---

Fix GitHub issue $ARGUMENTS following our coding standards.

1. Read the issue with `gh issue view $ARGUMENTS`
2. Search the codebase for relevant files
3. Implement the fix and write tests
4. Create a commit
```

Invoke it with `/fix-issue 42`. The `$ARGUMENTS` placeholder gets replaced with your input. Setting `disable-model-invocation: true` means only you can trigger it -- the agent won't decide to fix random issues on its own.

### Dynamic context

Skills can inject live data before the prompt reaches the agent. Use the `` !`command` `` syntax:

```yaml
---
name: pr-summary
description: Summarize the current PR
context: fork
---

PR diff: !`gh pr diff`
Changed files: !`gh pr diff --name-only`

Summarize this pull request concisely.
```

Those shell commands run first. The agent sees the actual PR data, not the commands. This is preprocessing, not something the agent executes.

### Skills across tools

OpenCode uses the same `SKILL.md` format and also reads from `.claude/skills/` for compatibility. Copilot takes a different approach -- reusable prompts live in `.github/copilot/prompts/*.md` and you reference them in chat with `#prompt-name`.

The concept is the same everywhere: package a workflow once, reuse it forever.

### Bundled skills

Claude Code ships with powerful built-in skills. `/batch` splits large changes across parallel worktrees. `/simplify` reviews your recent changes for quality issues. `/debug` enables debug logging and helps troubleshoot.

These are prompt-based -- they give the agent a detailed playbook and let it orchestrate the work using its tools.

### When to create a skill

Create a skill when you find yourself giving the same multi-step instructions more than twice. Deployment procedures, PR reviews, issue triage, code generation from templates -- anything with a repeatable pattern is a good candidate.

Keep `SKILL.md` focused. If your skill needs reference material, put it in supporting files within the skill directory and reference them from the main file.
