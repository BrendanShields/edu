Sometimes one agent isn't enough. You're deep in an implementation, and you need to understand how the billing module works -- but you don't want to pollute your current context with fifty files of research.

Subagents let you divide and conquer. Delegate the research to a specialist, keep your main conversation clean, and get a summary back.

### How subagents work

A subagent is an isolated AI assistant with its own context window. The main agent sends it a task, it works independently, and it returns results. Your main conversation never sees the intermediate steps -- just the conclusion.

Think of it like sending an intern to the library. They do the research, bring back a summary, and you keep working.

### Built-in subagents (Claude Code)

Claude Code ships with three: **Explore** is fast and read-only, powered by Haiku. It's great for searching codebases and answering questions. **Plan** handles research for planning mode. **General-purpose** has full tool access for complex multi-step tasks.

Claude decides when to delegate automatically. When it invokes Explore, it even specifies thoroughness -- quick, medium, or very thorough -- based on the complexity of the question.

### Built-in subagents (OpenCode)

OpenCode has a similar split. **General** handles multi-step tasks with full tool access and can run in parallel. **Explore** is fast, read-only codebase search. Invoke them manually with `@general` or `@explore`, or let the primary agent delegate automatically.

### Creating custom subagents

This is where it gets interesting. Define a subagent in `.claude/agents/` (Claude Code) or `.opencode/agents/` (OpenCode):

```markdown
---
name: security-reviewer
description: Reviews code for security vulnerabilities
tools: Read, Grep, Glob
model: sonnet
---

Analyze code for SQL injection, XSS, hardcoded secrets,
and missing input validation. Rate each finding by severity.
```

Now you have a security specialist you can invoke any time. Give it read-only tools so it can't accidentally change anything. Choose a faster model if speed matters more than depth.

### Parallel sessions with git worktrees

Subagents share the same repository. For truly parallel work -- where two agents might edit the same file -- use git worktrees.

Each worktree is an independent working copy. Start Claude Code in each one, and you have multiple agents working on different features simultaneously without file conflicts.

The built-in `/batch` skill automates this pattern. It decomposes a large task into units, spawns one agent per worktree, and each opens its own PR.

### When to delegate vs. keep inline

Delegate when the task is exploratory, independent of conversation context, or parallelizable. A codebase search, a security review, a documentation sweep -- these are perfect for subagents.

Keep work inline when you need back-and-forth discussion, when the task depends on recent conversation context, or when the result will immediately shape your next prompt. Subagents don't see your conversation history, so they're not great for tasks that need that context.
