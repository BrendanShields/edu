OpenCode is the tool you fully control. You choose the model, the provider, the rules. It's open source, so you can read the code, fork it, or contribute. Nothing is locked behind a vendor decision.

That level of control matters when you're choosing the foundation for your daily workflow.

### The TUI

OpenCode's terminal interface is its signature. It's not just a prompt in a box -- it's a full application with fuzzy file search (`@`), inline shell commands (`!`), slash commands, and agent switching.

Reference files with `@` and OpenCode pulls them into context. Prefix a command with `!` to run it without leaving the conversation. Hit `Tab` to cycle between agents. Type `/undo` if something goes wrong -- it reverts the code change and removes the message from history.

You might wonder why a TUI instead of an IDE extension. The answer is portability. OpenCode works the same on any machine, any OS, any terminal.

### Provider flexibility

This is OpenCode's superpower. Anthropic, OpenAI, Google, local models -- bring your own API key and go. No vendor lock-in, no single point of failure.

New to LLM providers? **Zen** is OpenCode's curated gateway. Run `/connect`, select "opencode", and sign in. Zen is a tested list of models the team has benchmarked for coding agent quality. You get good defaults without the research.

Or bring your own key: `opencode auth login` walks you through it.

### The agent system

OpenCode ships with two primary agents and two subagents. Think of primary agents as your main working modes, and subagents as specialists you call in.

**Build** is the default. Full tool access -- it reads, writes, and runs commands. **Plan** is read-only. It analyzes and suggests without changing anything. Switch between them with `Tab`.

For delegation, invoke subagents with `@`. **General** handles multi-step tasks with full tool access -- useful for parallel work. **Explore** is fast, read-only codebase search. Ask `@explore How does the payment module work?` and it digs through your code without modifying a thing.

### Custom agents

Want a security auditor that can only read files and run `git diff`? Define it in `.opencode/agents/review.md` with a markdown file and YAML frontmatter. Set the model, the permissions, the system prompt. Or run `opencode agent create` for an interactive setup.

Custom agents let you encode your team's expertise into reusable personas.

### When to choose OpenCode

Choose OpenCode when you want open source, provider flexibility, or full control over configuration. It shines for teams that don't want to commit to a single AI vendor, for developers who prefer a polished terminal workflow, and for anyone who values transparency in their tools.

If you need tight GitHub integration or IDE-first workflows, look at Copilot. If you want the deepest Anthropic integration, look at Claude Code. But if control is the priority, OpenCode is the answer.
