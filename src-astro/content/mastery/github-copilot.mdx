Copilot meets you where you already are -- inside your editor. While other tools ask you to switch to a terminal or a new app, Copilot is just there, woven into the place you write code every day.

That integration isn't just convenient. It fundamentally changes what AI assistance feels like.

### Inline completions

This is where most people start. You type a few characters, and Copilot suggests the rest. Accept with `Tab`, reject by keeping typing.

But it goes further than autocomplete. **Next Edit Suggestions** predict where your next change will be -- not just at the cursor, but elsewhere in the file. You're renaming a variable, and Copilot offers to update the other references before you get there.

Think of it as a pair programmer who reads ahead.

### Chat

When you need more than a suggestion, open Copilot Chat. Ask questions about your code, get explanations, brainstorm approaches. It works in the IDE, on GitHub.com, on mobile, and in the CLI.

Scope the conversation with participants like `@workspace` (VS Code) or `@project` (JetBrains) to give Copilot your full project as context. The difference between a vague answer and a precise one is usually context.

### Agent mode

Here's where Copilot enters the same territory as Claude Code and OpenCode. In agent mode, Copilot doesn't just suggest -- it acts.

Open Copilot Edits (`Cmd+Shift+I`), switch to Agent mode, and describe what you want. Copilot figures out which files to change, makes the edits, runs terminal commands, checks the results, and iterates. The same agentic loop you've seen elsewhere, but inside your editor.

The key difference from Edit mode: you don't pick the files. The agent does. You describe the goal, and Copilot works out the plan.

### The coding agent

What if the agent didn't need your editor at all? The **coding agent** works asynchronously on GitHub. Assign an issue to `@copilot`, and it spins up its own environment via GitHub Actions, implements the change, and opens a draft PR.

You review the PR like any other. Leave comments, and the coding agent iterates. It runs CodeQL for security, scans for secrets, and checks dependencies -- all built in.

This is the "fire and forget" workflow. Start it before lunch, review the PR after.

### Spaces

Copilot's answers are only as good as the context it has. **Spaces** let you curate that context -- bundle code, docs, specs, and design notes into a named collection. When you chat with Copilot in a Space, it grounds its answers in exactly the material that matters.

Think of Spaces as a reading list for your AI.

### Code review

Copilot also reviews pull requests. Request a review from Copilot the same way you'd request one from a teammate. It leaves suggestions, flags issues, and helps catch things humans miss on a Friday afternoon.

### When to choose Copilot

Copilot is the right choice when your workflow already lives on GitHub. Issues, PRs, Actions, code review -- Copilot touches all of them. It's also the only tool here with a true async background agent and inline completions in the same package.
